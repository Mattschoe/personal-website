Ledger

A plain-text, double-entry finance tracker that lives entirely in your terminal — built for people who want to own their financial data without a spreadsheet or a subscription.

View on GitHub ↗ Live demo ↗ Read the docs

Ledger started as a shell alias and a text file. I wanted to know where my money went without handing my bank logins to a startup, and every tool I tried wanted exactly that. So I built the smallest thing that could work: a command that appended a line to a file.

Three years later it's a proper double-entry accounting engine, but the philosophy hasn't moved. Your data is a human-readable file you can read in any editor, version with git, and grep through at 2am. The program just makes it pleasant.

The problem

Personal finance apps optimise for engagement, not understanding. They bury the ledger — the actual record of what happened — under charts designed to keep you scrolling. I wanted the opposite: the record front and centre, and reports I could trust because I could see exactly how they were computed.

If you can't grep it, you don't really own it.

How it works

Every transaction is a few lines of plain text. Ledger parses the file, validates that every entry balances, and builds an in-memory graph you can query — balances, cashflow, budgets — in milliseconds. SQLite is used only as a cache; delete it and nothing is lost.

What I learned

Writing a parser that gives genuinely helpful error messages is harder, and more rewarding, than the accounting maths. Most of the kind feedback I get isn't about features — it's about the moment Ledger tells someone exactly which line of their file is wrong, and why.
