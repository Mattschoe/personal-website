Matt's personal site — build spec

A personal site with three content streams — Recipes, Projects, and a Blog — and a Home page that surfaces the latest of each. Warm editorial aesthetic, built from one palette in two themes (Daylight & Twilight). This document is the source of truth; the sibling HTML files are the reference implementation.

01 — Recommendation

Suggested stack

The reference is plain HTML + one CSS file + a tiny theme script, with no build step — deliberately. It will drop straight into any framework. My recommendation for the real build:

  • Jetpack Compose
  • Markdown / MDX content collections — one folder per stream (recipes/, projects/, blog/). Front-matter drives the cards. See the content model below.
  • CSS custom properties for theming exactly as in styles.css — do not reach for a CSS framework, the token set is small and intentional.
  • The Home "latest" feed is a query across all three collections, sorted by date — not hand-maintained.
Note on themes

"Two color schemes" is implemented as one palette, two modes: Daylight (light/warm) is default; Twilight (dark) is opt-in via the header toggle and persists in localStorage. Both are documented below with exact tokens. There is no third "blue" scheme — the sketch's blue accent is replaced by the Grenadine/Beeswax accent from your palette.

Category color-coding

Each content type has a fixed accent used on its tag/pill: Recipe → Grenadine, Project → Sage, Essay → Beeswax. On Daylight the sage/beeswax are darkened (--c-project, --c-essay) for legible text on light paper; on Twilight the pure palette values are used.

05 — Spacing & form

Spacing, radius, shadow

Spacing scale

--s-1  .25rem    --s-6  2rem
--s-2  .5rem     --s-7  3rem
--s-3  .75rem    --s-8  4.5rem
--s-4  1rem      --s-9  7rem
--s-5  1.5rem

Sections use padding-block:--s-8. Always lay rows/groups out with flex/grid + gap, never margins between siblings.

Form language

  • Radius: small & crisp — 4px default, 8px cards. Pills use 999px. This is editorial, not bubbly.
  • Borders: 1.5px solid var(--line-strong) — drawn, confident lines are core to the look.
  • Shadow: hard offset 6px 6px 0 var(--line-strong) on hover (no blur) — a print/risograph feel. Soft blur shadows are avoided.
  • Hover: cards/buttons lift via translate(-3px,-3px) + hard shadow.
  • Container: 1180px max; reading column 680px.
06 — Components

Component inventory

All live in styles.css. Class names in mono.

  • .site-header — sticky, blurred, bottom hairline. Wordmark + nav + theme toggle. Collapses nav under 640px (build a menu button).
  • .kicker — mono uppercase eyebrow label.
  • .tag / .tag--recipe / --project / --essay — category pill with leading dot, colored by stream.
  • .btn / .btn--accent and .arrow-link — actions; arrow links use the ↗ glyph from the sketch.
  • .card — image + tag + title + excerpt; lifts on hover. .grid-3 / .grid-2 wrappers.
  • .featured — split hero card for the newest item on Home.
  • .ph — image placeholder. data-tone tints it (sage/beeswax/grenadine/latte); data-ph sets the corner caption. Replace each with a real <img> at the same aspect ratio.
  • .read — article column: lead paragraphs, blockquote with accent rule, .dropcap first-letter.
  • .stat-row / .ing-list / .steps — recipe-specific: stat strip, checkbox ingredient list, numbered method timeline.
  • .proj — alternating left/right project rows; .stack chips + .status dot.
  • .site-footer — 3-col, wordmark + link columns + bottom bar with theme toggle.
07 — Page anatomy

Pages

Home /index.html

Hero ("Hi! I'm Matt", tagline, circular portrait, quick-links + social — straight from your sketch) → Latest feed: one featured newest item + a 3-up mixed grid drawn across all streams, each card tagged by type → About strip (portrait + bio + CTAs).

The feed is a date-sorted merge of all three collections — show the most recent 4–7 items.

Recipes index /recipes/index.html

Page head + filter chips (category) → 3-up card grid. Cards show category + time + title + one-line excerpt.

Single recipe /recipes/recipe.html

Breadcrumb → split hero (intro + square photo) → stat strip (time / yield / effort) → two-column body: sticky ingredients list (left) + numbered method timeline (right) + "Matt's note" callout.

Projects index /projects/index.html

Page head → full-width alternating rows (image / text swap sides). Each: status dot, year, title, summary, stack chips, "case study →".

Single project /projects/project.html

Breadcrumb → big title + actions (GitHub / demo / docs) → wide screenshot → article + sticky spec rail (stack, role, status, year, numbers).

Blog index /blog/index.html

Page head → editorial post rows: date · title + excerpt · read-time. Whole row is the hover target.

Single post /blog/post.html

Centered post header (tag, title, byline) → lead figure → 680px reading column with dropcap → tag row → "keep reading" 2-up.

09 — Theming

How the theme toggle works

  • Default = Daylight. Twilight is applied by setting data-theme="dark" on <html>; all tokens remap via the [data-theme="dark"] block.
  • Choice persists in localStorage["matt-theme"].
  • A tiny inline script in each <head> applies the saved theme before paint to avoid a flash. theme.js wires every [data-theme-toggle] button (header + footer).
  • Optional improvement: default to prefers-color-scheme when no saved value exists.
  • All transitions are disabled under prefers-reduced-motion.
10 — Before you ship

Build notes & open items

  • Images: every .ph block is a placeholder — swap for a real <img> at the labelled aspect ratio. Matt to supply: portrait (square + 3:4), recipe photos (4:3 / square), project screenshots (16:8), post lead images (16:8).
  • Mobile nav: nav is hidden under 640px in the reference — implement a hamburger/menu drawer.
  • Social links + email are placeholders (#) — wire real URLs.
  • Recipe filters are visual only here — make them functional (filter by category tag).
  • SEO/meta: add per-page <meta> description, OpenGraph image (use the hero), and an RSS feed for the blog.
  • Accessibility: color pairs are chosen for contrast; keep focus styles visible when you templatise.
